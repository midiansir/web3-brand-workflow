## Eight Questions

1. What is the project called?
2. Who is it for?
3. What scene or problem does it address?
4. What action should the user take now?
5. What is already visible in the source materials?
6. What can be publicly written now?
7. What cannot enter public copy or public visuals?
8. What must this task deliver?

## Risk Categories

| Category | Text risk | Visual risk | Safer handling |
| --- | --- | --- | --- |
| Token and listing | token launch, claim, airdrop, listing, Binance, Gate.io, liquidity, price | Token icon, exchange logo, claim button, price chart | Remove unless official source permits exact wording. |
| Yield and rewards | APY, yield, bribes, fees, stable income, dividend, passive income | Coin piles, cash rain, yield chart, reward box | Replace with product action or official update guidance. |
| Financing and investor support | funded, investor-backed, top investors, capital support | Investor logo wall, bank building, funding poster | Keep in `待确认` until disclosure is confirmed. |
| Partnership and authorization | strategic partnership, official partner, authorized by | Partner wall, handshake, certificate, logo collage | Use only after authorization. |
| Launch and access | now live, public launch, open registration, whitelist, guaranteed access | Countdown, register button, eligibility pass | Use current stage language. |
| Audit and security | audited, certified, fully secure, zero downtime | Audit badge, shield certificate, compliance stamp | Use official safety reminders; avoid certification claims. |
| User scale and performance | thousands of users, TVL, volume, fastest-growing, global largest | Growth map, user counter, TVL chart | Require data source and date. |
| Wallet and data | wallet permissions, data tracking, user profile, analytics dashboard | Connect wallet hero, address table, user profiling UI | Keep abstract until product and privacy wording are confirmed. |
| RWA and financial assets | asset-backed, RWA yield, tokenized assets, investment product | Real estate, gold, bonds, bank assets | Use process or scenario language only after compliance review. |
| Event and sales pressure | sign today, limited seats, payment counter, onsite deal | Cash register, contract signing, countdown | Use information and consultation language. |

## Narrative Extraction

Build the narrative through this ladder:

```text
Raw source fact
-> Confirmed public fact
-> User pain or desired state
-> Project's specific role
-> Proof or visible asset
-> Current-stage CTA
```

Example structure:

| Layer | Question | Output |
| --- | --- | --- |
| Fact | What can we prove? | Source-backed statement |
| Problem | What is currently confusing, fragmented, risky, or slow? | Plain user pain |
| Role | What does the project do in that scene? | Clear project position |
| Difference | Why is this project not a generic Web3 tool? | Track-specific distinction |
| Trust | What is visible now? | Existing product, asset, document, role, team-approved proof |
| Boundary | What remains unconfirmed? | `待确认` and safe CTA |

## Brand Archetype Use

Use archetypes as practical production aids, not as decorative labels:

| Archetype | Use when | Language tendency | Visual tendency |
| --- | --- | --- | --- |
| Operator | Workflow, campaign, community, SaaS, dashboards | Clear, modular, task-focused | Cards, paths, entry hubs, status tags |
| Builder | Infra, L1/L2, protocol, devtools | Precise, technical, evidence-led | Architecture, code, nodes, diagrams |
| Market native | DeFi, DEX, liquidity | Direct, mechanism-aware, risk-clean | Product panels, liquidity routes, dark UI |
| World maker | GameFi, IP, NFT, community culture | Story and character first | Character systems, worlds, artifacts |
| Institution | DAO, foundation, launchpad, ecosystem | Stable, official, governance-aware | Release hubs, governance flows, formal diagrams |

## Output Safety Rules

- Keep every unsupported claim out of public copy.
- Move unclear facts into `待确认`, not into confident prose.
- In multi-language work, do not let English or other languages add stronger claims than the Chinese base.
- In visuals, avoid risk through image elements as well as text.
- Treat old rejected assets, old project names, and archived attempts as historical unless current files say otherwise.
